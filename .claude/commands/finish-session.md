# Finish Session

用于结束一次非琐碎开发会话，保证下一轮可以直接接手。

## 检查

1. 标准验证是否运行；如果未运行，记录原因。
2. 功能状态是否真实；没有证据不能标记 `passing`。
3. 是否跑过足够层级的验证：单元、集成、端到端或明确说明为什么不能跑。
4. 是否清理临时调试代码、无主 TODO、注释掉的废代码和一次性日志。
5. 是否存在未记录的半成品、blocker 或风险。
6. `tasks/progress.md` 是否需要更新。
7. `tasks/feature-list.json` 是否需要更新。
8. 长会话或跨模块任务是否需要更新 `tasks/session-handoff.md`。
9. 重要改动后是否需要更新 `tasks/quality.md`。

## 输出

- 已完成
- 运行过的验证
- 证据
- 清洁状态检查
- 未验证或风险
- 下一步最佳动作
