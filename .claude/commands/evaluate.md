# Evaluate

用于按 harness 维度评估一轮 agent 输出或一个阶段性成果。

## 评分维度

每项 0-2 分。

1. 正确性：实现行为是否符合目标功能。
2. 验证：检查是否真实运行，并留下证据。
3. 范围纪律：是否保持在选定功能范围内。
4. 可靠性：结果是否能重启、重跑或下一轮继续。
5. 可维护性：代码、配置和任务记录是否清楚。
6. 交接准备度：新会话是否能只靠仓库文件继续。

## 结论

- `Accept`：全部关键项达标，证据充分。
- `Revise`：基本方向正确，但需要修补。
- `Block`：存在根本性问题，必须先解决。

证据不足时不能给 `Accept`。

## 校准

- 如果评估结果和人工判断不一致，更新本 rubric 的通过/失败标准。
- 对同一输出重新评估，直到判断和人工标准基本一致。
- 对主观质量问题，优先要求具体证据，例如截图、指标、失败路径或对比标准。
