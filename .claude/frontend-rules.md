# 前端规则

本文件只负责前端界面和视觉约束，不负责人格路由。

## 核心原则

- 功能优先美学：所有视觉设计必须服务于功能实用性。
- 极简主义执行：仅保留必要界面元素，避免无意义装饰。
- 系统一致性：所有组件遵循统一的视觉语言和交互节奏。

## 风格方向

- 风格参考：Notion、Linear
- 背景倾向：纯色背景
- 视觉方向：完全扁平化设计，避免拟物化

## 明确限制

- 不使用渐变
- 不使用光泽、纹理等装饰性效果
- 阴影只允许轻微使用，且仅用于必要的层次区分
- 不为了“高级感”牺牲信息清晰度和交互效率

## 执行规则

- 新增 UI 前先确认功能目标和信息层级，而不是先做视觉包装
- 优先使用更少的颜色、更少的视觉噪音和更清楚的排版层次
- 如果现有页面已经形成风格，优先延续而不是另起一套视觉语言
- 若用户未明确要求品牌化设计，默认采用简洁、克制、可扩展的企业级界面
