# Clean State Check

用于会话结束前检查仓库是否处于下一轮可以直接接手的状态。

## 检查项

1. 构建或基础验证状态清楚。
2. 测试、lint、类型检查或替代验证结果已记录。
3. `tasks/progress.md` 和 `tasks/feature-list.json` 反映真实状态。
4. 没有未记录的半成品、临时文件、调试日志、注释掉的废代码或无主 TODO。
5. 标准启动路径和标准验证路径没有被破坏。
6. 下一步最佳动作明确。

## 结论

- `Clean`：下一轮可直接继续。
- `Needs Record`：代码状态可接受，但任务记录缺失。
- `Dirty`：存在半成品、验证失败或不可接手状态。
